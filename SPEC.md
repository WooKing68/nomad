# K-Nomad 웹사이트 개선 사양서

## 프로젝트 개요
K-Nomad 플랫폼을 도시 추천 시스템으로 개선하기 위한 단계별 개발 계획입니다. 각 Phase는 독립적으로 실행 가능하며 점진적으로 프로젝트를 개선해나갑니다.

## 현재 상태 분석
- **기존 페이지**: 홈(`/`), 로그인(`/login`), 회원가입(`/signup`), 대시보드(`/dashboard`), 에러 페이지(`/error`)
- **현재 기능**: 간단한 K-Nomad 플랫폼 랜딩 페이지, Supabase 인증 시스템

## 개발 원칙
- [ ] 데이터베이스 사용하지 않고 가짜 데이터로 진행
- [ ] 각 단계는 독립적으로 실행 가능한 형태로 완성
- [ ] 점진적 개선을 통한 안정적인 개발

---

## Phase 1: 기본 구조 정리 및 불필요한 페이지 제거

### 이번 단계에서 개선할 사항 오버뷰
홈페이지와 인증 관련 페이지를 제외한 모든 페이지를 삭제하고, 기본적인 도시 추천 시스템의 토대를 마련합니다. 현재의 단순한 랜딩 페이지를 도시 리스트를 보여주는 메인 페이지로 변경합니다.

### 수정/개선해야 하는 기능
- [x] `/dashboard` 페이지 삭제
- [x] `/error` 페이지 삭제
- [x] 홈페이지(`/`)를 도시 리스트 페이지로 변경
- [x] 기본적인 도시 카드 컴포넌트 구조 생성
- [x] 임시 가짜 데이터 생성 (최소 5-6개 도시)
- [x] "도시 리스트" 제목으로 페이지 헤더 변경

### 작업 완료 후 검증/확인해야 하는 기능
- [x] 홈페이지에서 도시 리스트가 정상적으로 표시되는지 확인
- [x] 삭제된 페이지들(`/dashboard`, `/error`)이 404를 반환하는지 확인
- [x] 로그인/회원가입 페이지가 정상 작동하는지 확인
- [x] 각 도시 카드가 기본 정보를 표시하는지 확인

## Phase 2: 좋아요/싫어요 시스템 구현

### 이번 단계에서 개선할 사항 오버뷰
별점 평점 시스템을 제거하고 좋아요/싫어요 버튼으로 대체합니다. 사용자가 버튼을 클릭할 때 상태가 변경되고 숫자가 업데이트되는 인터랙티브한 기능을 구현합니다.

### 수정/개선해야 하는 기능
- [x] 기존 별점 평점 시스템 제거 (있다면)
- [x] 좋아요/싫어요 버튼 컴포넌트 생성
- [x] 버튼 클릭 시 아이콘 색상 변경 로직 구현
- [x] 좋아요/싫어요 숫자 카운트 기능 구현
- [x] 각 도시 데이터에 좋아요/싫어요 수치 추가
- [x] 로컬 스토리지를 이용한 사용자 좋아요 상태 저장

### 작업 완료 후 검증/확인해야 하는 기능
- [x] 좋아요 버튼 클릭 시 아이콘이 활성화 색상으로 변경되는지 확인
- [x] 싫어요 버튼 클릭 시 아이콘이 활성화 색상으로 변경되는지 확인
- [x] 버튼 클릭 시 해당 카운트 숫자가 증감하는지 확인
- [x] 페이지 새로고침 후에도 사용자의 좋아요 상태가 유지되는지 확인
- [x] 좋아요와 싫어요가 상호 배타적으로 작동하는지 확인

## Phase 3: 네비게이션 정리 및 필터 시스템 기초

불필요한 네비게이션 링크를 제거하고 인증 관련 버튼만 남깁니다. 도시 필터링을 위한 기본 UI 구조를 구현합니다.

### 수정/개선해야 하는 기능
- [x] 네비게이션 바에서 대시보드 등 불필요한 링크 제거
- [x] 로그인/로그아웃, 프로필 관련 버튼만 유지
- [x] 필터 컴포넌트 기본 구조 생성
- [x] 예산 필터 UI 구현 (100만원, 100~200만원, 200만원)
- [x] 지역 필터 UI 구현 (전체, 수도권, 경상도, 전라도, 강원도, 제주도, 충청도)
- [x] 환경 필터 UI 구현 (자연친화, 도심선호, 카페작업, 코워킹 필수)
- [x] 계절 필터 UI 구현 (봄, 여름, 가을, 겨울)

### 작업 완료 후 검증/확인해야 하는 기능
- [x] 네비게이션에서 불필요한 링크가 제거되었는지 확인
- [x] 인증 관련 버튼들이 정상 작동하는지 확인
- [x] 모든 필터 카테고리가 UI에 표시되는지 확인
- [x] 필터 옵션들이 선택 가능한지 확인
- [x] 필터 선택 시 시각적 피드백이 있는지 확인

- [x] 예산, 지역, 환경, 최고 계절 정보를 Key-Value로 표시
- [x] 각 필터 정보에 대한 시각적 태그/배지 디자인
- [x] 기존 가짜 데이터에 모든 필터 속성 추가
- [x] 각 도시별로 최소 하나의 필터 요소 포함 확인
- [x] 카드 내 정보 레이아웃 최적화

### 작업 완료 후 검증/확인해야 하는 기능
- [x] 모든 도시 카드에 예산 정보가 표시되는지 확인
