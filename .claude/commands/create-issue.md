# GitHub 이슈 생성기

GitHub 이슈를 체계적으로 생성하는 커스텀 프롬프트입니다.

## 입력된 이슈
**생성할 이슈:** $ARGUMENT

---

## 이슈 분석 및 구조화

### 1. 이슈 분류
- [ ] **Bug Report** (버그 신고)
- [ ] **Feature Request** (기능 요청)
- [ ] **Enhancement** (기능 개선)
- [ ] **Documentation** (문서화)
- [ ] **Question** (질문)
- [ ] **Task** (작업)

### 2. 우선순위 설정
- [ ] **Critical** (즉시 수정 필요)
- [ ] **High** (높음)
- [ ] **Medium** (보통)
- [ ] **Low** (낮음)

### 3. 예상 복잡도
- [ ] **XS** (1-2시간)
- [ ] **S** (반나절)
- [ ] **M** (1-2일)
- [ ] **L** (1주)
- [ ] **XL** (1주 이상)

---

## GitHub 이슈 템플릿

### 제목
```
[분류] 간결한 이슈 제목 (동사로 시작)
```

### 본문

#### 📋 요약
간단한 한 문장으로 이슈의 핵심 내용을 설명합니다.

#### 🎯 목표
이 이슈를 해결함으로써 달성하고자 하는 목표를 명시합니다.

#### 📝 상세 설명
[이슈 유형별 상세 내용]

**Bug Report인 경우:**
- **현재 동작**: 현재 어떻게 동작하는지
- **예상 동작**: 어떻게 동작해야 하는지
- **재현 단계**:
  1. 단계 1
  2. 단계 2
  3. 단계 3
- **환경 정보**: OS, 브라우저, 버전 등

**Feature Request인 경우:**
- **사용자 스토리**: "~로서, ~을 위해, ~하고 싶다"
- **비즈니스 가치**: 이 기능이 제공하는 가치
- **사용 시나리오**: 구체적인 사용 예시

**Enhancement인 경우:**
- **현재 상태**: 기존 기능의 한계점
- **개선 방향**: 어떻게 개선할 것인지
- **기대 효과**: 개선 후 얻을 수 있는 이점

#### ✅ 완료 조건 (Definition of Done)
- [ ] 조건 1
- [ ] 조건 2
- [ ] 조건 3
- [ ] 테스트 케이스 작성 및 통과
- [ ] 코드 리뷰 완료
- [ ] 문서 업데이트 (필요시)

#### 🔧 구현 힌트 (선택사항)
- 기술적 접근 방법이나 참고 자료
- 관련 파일이나 컴포넌트
- 고려사항이나 제약사항

#### 📎 관련 자료
- [ ] 관련 이슈: #번호
- [ ] 참고 문서: [링크](URL)
- [ ] 디자인 시안: [링크](URL)
- [ ] API 문서: [링크](URL)

#### 🏷️ 라벨
`bug`, `feature`, `enhancement`, `documentation`, `good first issue`, `help wanted`, `priority-high`, `size-medium`

#### 👥 담당자
- **Assignee**: @username
- **Reviewer**: @username

## 이슈 생성 체크리스트

### 생성 전 확인사항
- [ ] 중복 이슈가 없는지 검색했는가?
- [ ] 제목이 명확하고 구체적인가?
- [ ] 완료 조건이 측정 가능한가?
- [ ] 적절한 라벨을 선택했는가?
- [ ] 우선순위가 적절한가?

### 생성 후 작업
- [ ] 관련 팀원에게 멘션 또는 알림
- [ ] 프로젝트 보드에 추가 (해당시)
- [ ] 마일스톤 설정 (해당시)
- [ ] 관련 이슈와 연결

---

## 이슈 작성 가이드라인

### 좋은 이슈 제목 예시
- ✅ `[Bug] 로그인 후 대시보드 페이지가 로드되지 않음`
- ✅ `[Feature] 사용자 프로필 이미지 업로드 기능 추가`
- ✅ `[Enhancement] 검색 결과 페이지 성능 개선`

### 피해야 할 이슈 제목 예시
- ❌ `버그 있음`
- ❌ `기능 추가`
- ❌ `이거 고쳐주세요`

### 작성 팁
1. **구체적으로**: 모호한 표현보다는 구체적인 설명
2. **객관적으로**: 개인 의견보다는 사실 기반 서술
3. **액션 지향적으로**: 무엇을 해야 하는지 명확히 제시
4. **측정 가능하게**: 완료 여부를 판단할 수 있는 기준 제시

---

이 템플릿을 사용하여 `$ARGUMENT`에 생성하고 싶은 이슈 내용을 입력하면, 체계적이고 완성도 높은 GitHub 이슈를 생성할 수 있습니다.
